This drop-down card enables access to bookmarks. Bookmarks represent a snapshot of a session.  
Since an analysis may span several sessions, you are able to lodge and restore the past state of an analysis. Bookmarks are public and can be shared between users of that server.


#### Goal
The goal of this card is to enable bookmarked sessions to be continued.

This bookmark picker shows past bookmarks (in time order) and the dialogue allows:

 * Lodging of bookmarks
 * Renaming of bookmarks
 * Deleting of bookmarks
 * Restoring of bookmarks


The default name of a bookmark to be lodged is the name of the dataset in use.
If you wish to avoid overwriting bookamrks, then suffix the bookmark name with something distinct e.g. 'PatientData-modelling'. The name used must be valid as a folder name. Avoid the **/** character.

Timestamps help to distinguish similar named bookmarks. The latest bookmarks are at the top of the pick list.
Bookmarks can be large. Several copies of the data may be saved to the disk. For this reason, overwrite bookmarks whenever appropriate and delete obsolete bookmarks.  

Beware: bookmarks are **not** automatically saved when you close the session.
